Message: E5100

OTL Coverage: The Format field is invalid

There are two formats for Coverage tables:

Coverage Format 1 consists of a count of covered glyphs (GlyphCount), followed by an array of glyph indices (GlyphArray). The glyph indices must be in numerical order for binary searching of the list.

Coverage Format 2 consists of a count of glyph index ranges (RangeCount), followed by an array of records (RangeRecords). Each RangeRecord consists of a start glyph index (Start), an end glyph index (End), and the Coverage Index associated with the range's Start glyph. Ranges must be in GlyphID order, and they must be distinct, with no overlapping.

For more information, see the OTL common table formats.
