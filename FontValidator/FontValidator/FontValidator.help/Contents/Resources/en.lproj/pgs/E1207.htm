Message: E1207

hdmx: The hdmx width doesn't match the calculated width

The hdmx width (the pixel width of a glyph in ppem) does not match the value calculated by the current version of the rasterizer. These values may not match if glyphs have been updated but the hdmx table has not yet been recalculated (using a tool such as Microsoft's CacheTT). This problem may also occur if the hdmx table was generated using a previous version of the rasterizer, or another tool that performs the width calculations differently. To make the values match, use the latest version of CacheTT to calculate the cached values using the current rasterizer. However, you may not want to change these values if there are concerns about the way text in legacy documents might reflow.

For more information, see the hdmx table specification.
