Message: E1804

LTSH: The LTSH yPel value doesn't match the calculated value

The yPel value, the ppem for each glyph at which the scaling becomes linear, does not match the value calculated by the current version of the rasterizer. These values may not match if the LTSH table was generated using a previous version of the rasterizer, or another tool that performs the yPel calculations differently. To make the values match, use the latest version of CacheTT to calculate the cached values using the current rasterizer. However, you may not want to change these values if there are concerns about the way text in legacy documents might reflow.

For more information, see the LTSH table specification.
