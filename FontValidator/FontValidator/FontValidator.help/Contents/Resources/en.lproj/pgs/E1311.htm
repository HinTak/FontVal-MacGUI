Message: E1311

head: The macStyle bold bit doesn't match the name table's font subfamily string

There is a discrepancy between the setting of the bold bit of the macStyle field and the Subfamily name field in the name table. There are several places within the font file that the typeface may be characterized as 'Regular,' 'Bold,' 'Italic,' 'Condensed,' etc. Font designers should encode this information consistently throughout the font. Font clients use the information to:

  • organize related typefaces into families (i.e., Arial Narrow, Arial, Arial Bold, Arial Expanded...)
  • use appropriate names when enumerating typefaces to the user
  • obtain a font from the system according to certain attributes (weight, style, width)
The fields related to encoding this information exist in several different tables:
  • In the name table, Name ID strings (family name, subfamily name, full-font name, preferred family grouping, and preferred subfamily grouping)
  • In the OS/2 table, usWeightClass, usWidthClass, PANOSE(tm), and fsSelection.
  • In the head table, macStyle bits
Correct this data using a table editing tool.

For more information, see the head table specification.
