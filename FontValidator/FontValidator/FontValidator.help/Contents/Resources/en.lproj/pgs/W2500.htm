Message: W2500

VDMX: The VDMX data doesn't match the data calculated by the rasterizer

The VDMX data does not match the value calculated by the current version of the rasterizer. This data may not match if glyphs have been updated but the VDMX table has not been recalculated (using a tool such as Microsoft's CacheTT). The problem may also occur if the VDMX table was generated using a previous version of the rasterizer, or another tool that performs the calculations differently. To correct the data, use the latest version of CacheTT to calculate the cached values using the current rasterizer. However, you may not want to change these values if there are concerns about the way text in legacy documents might reflow.

For more information, see the VDMX table specification.
