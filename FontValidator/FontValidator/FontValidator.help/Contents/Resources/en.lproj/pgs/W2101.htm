Message: W2101

OS/2: A CodePage bit is set in ulCodePageRange, but the font is missing some of the printable characters from that codepage

If a bit is set (1) indicating codepage coverage, then font clients will consider the font fully functional for that codepage. Although a font designer may choose to set a codepage bit if the font is 'substantially' functional for a codepage, this may cause problems to operating systems or applications expecting full codepage coverage. Font designers should select character sets that support fully functional code pages, if at all possible.

For more information, see the OS/2 table specification.
