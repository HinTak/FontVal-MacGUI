Message: E2133

OS/2: The usWeightClass is not a valid value

The usWeightClass field indicates the visual weight of the characters in the font. Values may range from 100 (Thin) to 900 (Black), in increments of 100. The usWeightClass is used by the system to obtain a font when an application requests a font with a specified weight class (through the CreateFont API).
There are several places within the font file that the typeface may be characterized as 'Regular,' 'Bold,' 'Italic,' 'Condensed,' etc. Font designers should encode this information consistently throughout the font. Font clients use the information to:

  • organize related typefaces into families (i.e., Arial Narrow, Arial, Arial Bold, Arial Expanded...)
  • use appropriate names when enumerating typefaces to the user
  • obtain a font from the system according to certain attributes (weight, style, width)
The fields related to encoding this information exist in several different tables:
  • In the name table, Name ID strings (family name, subfamily name, full-font name, preferred family grouping, and preferred subfamily grouping)
  • In the OS/2 table, usWeightClass, usWidthClass, fsSelection, and PANOSE(tm)
  • In the head table, macStyle bits
Correct this data using a table editing tool.
For examples of how to use these various fields, see these name string examples.

For more information, see the OS/2 table specification.
