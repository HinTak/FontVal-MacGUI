Message: E2129

OS/2: The usDefaultChar field is nonzero, but the character is not mapped to a glyf

Whenever a request is made for a character that is not in the font, Windows provides this default character. If the value of this field is zero, glyph ID 0 is to be used for the default character. If the value of this field is non-zero, then the value of the field indicates the Unicode encoding of the glyph that Windows uses as the default character. This specified glyph must exist in the font.

For more information, see the OS/2 table specification.
