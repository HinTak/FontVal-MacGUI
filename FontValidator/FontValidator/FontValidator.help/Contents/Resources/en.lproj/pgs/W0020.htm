Message: W0020

Font: Tables are not in optimal order

OpenType fonts with TrueType outlines are most efficiently utilized by Windows when the tables are ordered as follows (from first to last):
head, hhea, maxp, OS/2, hmtx, LTSH, VDMX, hdmx, cmap, fpgm, prep, cvt, loca, glyf, kern, name, post, gasp, PCLT, DSIG

The initial loading of an OpenType font containing CFF data will be more efficient if the following table ordering is used within the body of the font (from first to last):
head, hhea, maxp, OS/2, name, cmap, post, CFF, other tables (as convenient)

This efficiency results from the operating system being able to access commonly used information without having to move between 'pages' of data in memory. Therefore, the efficiency gained by Windows applies to implementations of the rasterizer on other operating systems that use a paging architecture.

To reorder the tables of a font, use the FastFont tool (available from Microsoft). You will need to create a text file containing table tags in the order in which you want them, and then reference this text file as a parameter of the FastFont tool. (The default order in which tables are placed by FastFont is antiquated.) Tables not included in the list above should be placed at the end of the table before the font is signed (and a DSIG table is appended as the last table in the font). If two tables are dependent on each other (like vhea/vmtx) then they should be placed next to each other.

For more information, see the Recommendations for OpenType fonts.
